## 1. Goal

Let users import their own curated icons as a **local `.zip` pack** instead of
uploading one custom icon at a time, while treating the archive as fully untrusted
input. No bundled third-party packs, no brand logos in the repo, no remote/URL
importing, SVG stays rejected, no Docker/deployment or auth change.

## 2. Approved scope (single coherent phase)

Backend import/validation/storage + API + a picker **Packs** tab + pack icon
serving + pack deletion, delivered together (the UI is a near-clone of the existing
Custom tab, so splitting added no value).

v0.2.8 stays: import zip pack ‚Üí validate manifest/assets ‚Üí store pack ‚Üí list pack
icons ‚Üí Packs tab in the icon picker ‚Üí **manual** icon selection ‚Üí delete pack ‚Üí
safe fallback to initials.

**Explicitly out of scope / deferred:** **bulk icon-to-app matching** (see the
v0.2.9 future phase below), remote/URL pack fetching, bundled packs, brand logos,
SVG, an icon marketplace, pack-driven auto-suggestion (manifest `aliases`), and a
pack replace/upgrade flow (re-import is delete-first today).

### Future phase ‚Äî v0.2.9 ‚Äî Icon Pack App Matching (documented, not built)
A later phase may, **after** a pack is imported, add a review/matching UI that:
- suggests matches between existing apps and imported icon keys;
- shows a confidence/reason per suggested match where useful;
- lets the user tick/untick suggested matches;
- lets the user manually adjust suggestions before applying;
- **bulk-applies only the selected assignments after explicit confirmation** ‚Äî
  icons are **never auto-applied without user approval**.

v0.2.8's manual selection from the Packs tab is unaffected and remains the shipped
behaviour. Not started in v0.2.8.

## 3. Architecture as built

### Reference grammar
- `pack:<packId>/<iconKey>` (+ optional `?v=<variant>`), parsed additively in
  `parseIconRef` after `builtin:`/`custom:`, before the legacy fallthrough ‚Äî
  back-compatible, no apps-table change.
- `packId`/`iconKey` slugs: `^[a-z0-9]+(?:-[a-z0-9]+)*$`, ‚â§ 64; `variant` a
  free-form slug (‚â§ 32). Pack icons resolve to `mode:"img"` **only** (never mask),
  served from `/api/icons/packs/<packId>/<iconKey>/raw` (+ `?v=`). Invalid slugs or
  a missing pack/icon ‚Üí initials; an absent variant ‚Üí base icon.

### Manifest (`manifest.json`, `manifestVersion: 1`)
`id`, `name`, `version`, optional `author`/`license`/`homepage` (http(s),
display/link only ‚Äî never fetched), and `icons[]` of `{ key, label?, file,
variants? }`. zod-validated: slug id/keys, unique keys, `assets/`-rooted safe file
paths (no `..`, absolute, backslash, drive letter, NUL).

### ZIP safety (cheapest checks first)
1. Total zip-size cap (5 MB) before parsing.
2. Central-directory scan: entry-count cap (‚â§ 600; rejects the zip64 sentinel) and
   rejects symlinks/special files via Unix `st_mode` in external attributes.
3. `fflate` `filter`: `manifest.json` + `assets/` allowlist; per-entry and running
   **declared** uncompressed caps (zip-bomb guard); directory entries skipped.
4. Manifest JSON parse + zod validation.
5. Nested archives rejected by effect (bytes never sniff as PNG/WebP) and by the
   allowlist; missing referenced files rejected.

### Asset validation
Magic-byte sniff (PNG/WebP only ‚Äî SVG/GIF/JPEG/ICO/empty/malformed rejected;
declared `mime` advisory); 512 KB/asset, 16 MB/pack; **declared and actual** sizes
both validated; sha256 dedup (`<sha256>.<ext>`).

### Storage & DB
Files under `<ICONS_DIR>/packs/<packId>/<sha256>.<ext>` (no new env var); a
filesystem path is never exposed. **Atomic** import: stage ‚Üí `rename` ‚Üí txn rows;
any failure persists nothing and cleans staged files. Two additive tables
(`icon_packs`, `icon_pack_icons`, FK cascade, unique `(pack_id, key, variant)`) in
migration `0004`; no change to `apps`/`custom_icons`.

### API & serving
`protectedRoute` (auth + same-origin CSRF) on all of:
`GET /api/icons/packs`, `POST /api/icons/packs` (multipart `.zip`; duplicate id ‚Üí
`409 duplicate_pack`), `DELETE /api/icons/packs/[packId]`, and
`GET /api/icons/packs/[packId]/[iconKey]/raw` (+ `?v=`) with the v0.2.6 safe
headers (`nosniff`, inline, immutable).

### UI
A **Packs** tab in the icon picker: import a `.zip`, browse packs grouped by pack
with name/version/author/license and a display-only homepage link (never fetched),
select pack icons, delete a pack. Library/Custom/URL/None flows unchanged.

### Dependency decision
Added **`fflate`** (MIT, ~8 KB, zero runtime deps) ‚Äî the project had no ZIP parser
and Node's `zlib` is raw DEFLATE only; hand-rolling a parser for untrusted input is
its own risk. `fflate`'s `filter` exposes declared sizes pre-inflation; a small
hand-written central-directory scan adds entry-count bounding + symlink detection
that `fflate` does not surface.
