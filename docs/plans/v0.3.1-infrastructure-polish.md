# v0.3.1 — Infrastructure Polish & Runtime Display Fixes

**Branch:** `fix/v0.3.1-infrastructure-polish`
**Base:** `main` @ v0.3.0 (`efd6307`)

## Goal

A small, safe polish/fix pass over the v0.3.0 Infrastructure page. Improve display
quality, partial-telemetry behaviour, storage filtering, and minor formatting —
**without** adding new telemetry providers or changing backend architecture.

This is **display + filtering only**. No new providers (TrueNAS / SMART / NVIDIA /
CPU power / packet capture), no Docker/Dockerfile/compose change, no schema change,
no data-volume permission fix in code, no new dependency.

## Observed v0.3.0 issues addressed

| # | Issue | Fix |
|---|-------|-----|
| 1 | CPU clock can show `0 MHz` on limited hosts | Normalise `0`/non-finite clock to `null` server-side; `formatClockMhz` renders invalid as `—` |
| 2 | Invalid/unknown metrics could render as bogus values | Hardware-card `dash` helper treats `NaN`/`Infinity` as `—` (a real `0` is still shown) |
| 3 | Storage shows container internals (`resolv.conf`, etc.) | `parseMounts` filters internal mountpoints by mountpoint (they inherit a real fstype, so the allowlist can't catch them) |
| 4 | App data volume looks like NAS/pool storage | The filesystem holding the app data dir is flagged `isAppData` and badged **App data** |
| 5 | NAS pool/SMART unavailable looks like a gap/failure | Calm Storage note + clearer "not configured — planned" TrueNAS copy |
| 6 | Page takes ~9s before all telemetry appears | Calm loading skeleton + overall wall-clock budget on per-container stats |

## Changes

### Server (`src/server/telemetry/`)
- **`proc.ts`**
  - `parseMounts` now drops container/internal bind-mounts via `isInternalMount`:
    explicit single-file config mounts (`/etc/resolv.conf`, `/etc/hosts`,
    `/etc/hostname`, `/dev/termination-log`) and runtime/pseudo prefixes
    (`/proc/`, `/sys/`, `/dev/`, `/run/`, `/var/run/`). Pseudo *fstypes* are still
    excluded by the existing `REAL_FSTYPES` allowlist.
  - New pure helpers `isInternalMount` and `pickAppDataMount` (exported, unit-tested).
- **`service.ts`**
  - CPU `clockMhz` normalised with `positiveFiniteOrNull` (`0`/`NaN`/`Infinity` → `null`).
  - `buildStoragePools` resolves the app data dir from `env.DATABASE_PATH`, uses
    `pickAppDataMount` to flag the holding filesystem, and sets `isAppData`.
  - Telemetry-source `detail` copy clarified: Local/Docker read as **Connected**;
    TrueNAS as **not configured — planned**.

### Types (`src/lib/types/infrastructure.ts`)
- `StoragePoolDTO` gains `isAppData: boolean` (transport-safe).

### Frontend
- **`src/lib/format.ts`** — `formatClockMhz` accepts `number | null | undefined` and
  returns `—` for invalid input (`null`, `0`, negative, `NaN`, `Infinity`).
- **`components/infrastructure/hardware-cards.tsx`** — hardened `dash` helper;
  CPU clock uses `formatClockMhz` directly.
- **`components/infrastructure/storage-overview.tsx`** — **App data** badge +
  "App / container storage" label for the app-data pool; calm footer note that NAS
  pool/SMART health appears once a NAS source (e.g. TrueNAS) is connected.
- **`components/infrastructure/infrastructure-dashboard.tsx`** — first-load
  `InfrastructureSkeleton` (layout-matching, `aria-busy`) replaces the bare spinner.
- **`server/telemetry/container-stats.ts`** — `STATS_BUDGET_MS` (2.5s) overall
  budget; workers stop pulling new containers past the budget and return a partial,
  best-effort result.

### Tests
- `tests/unit/format.test.ts` — `formatClockMhz` valid + invalid cases.
- `tests/unit/telemetry-parsers.test.ts` — `parseMounts` internal-mount filtering;
  `isInternalMount`; `pickAppDataMount`.

## Security / redaction
Unchanged. No raw host paths, socket paths, serials, stack traces, or backend
details reach the client. The app data dir is only used server-side to *flag* the
holding mount; only the existing redacted pool label (basename) is sent.

## Out of scope / future
Read-only **TrueNAS telemetry provider** (real NAS pool health, datasets, drive
SMART status, masked serials) is a later phase — not implemented here.
