# v0.2.9 — Icon Pack App Matching (implementation report)

Branch `feature/v0.2.9-icon-pack-app-matching` (from `main` at `v0.2.8.1`).
See `v0.2.9-icon-pack-app-matching-plan.md` and `docs/adr/0016-icon-pack-app-matching.md`.

## What shipped
A review screen (`/apps/icons`) that **suggests** pack-icon → app matches and
**bulk-applies only the ticked, confirmed selection**. Icons are never
auto-applied; the server independently enforces overwrite protection.

### Code
- **`src/lib/icons/match-core.ts`** (new) — `normalize`, `safeHost`,
  `strippedName`, `scoreAlias`, `QUALIFIER_TOKENS`, extracted verbatim from
  `suggest.ts`. Shared by the suggester and the matcher.
- **`src/lib/icons/suggest.ts`** — now imports the core; `suggestIconKey`
  behaviour unchanged (regression-guarded).
- **`src/lib/icons/pack-match.ts`** (new) — pure `matchPackToApps(apps, pack)`:
  per-app suggestion with `confidence` (high/medium/low), `reason`,
  `currentIsCustomised`, `alreadySet`, `defaultSelected`. Bidirectional qualifier
  stripping.
- **`src/lib/types/icons.ts`** — `PackMatchOutcomeDTO`, `PackMatchApplyResultDTO`.
- **`src/lib/validation/icons.ts`** — `packMatchApplySchema`.
- **`src/server/services/icon-packs.ts`** — `applyPackMatches(packId, input)`:
  pack-existence (→ null/404), icon-key/variant existence, variant→base fallback,
  app existence, already-set no-op, overwrite gate, partial-success per-item
  outcomes.
- **`src/app/api/icons/packs/[packId]/apply/route.ts`** (new) — `protectedRoute`
  POST.
- **`src/hooks/icon-packs-api.ts`** — `applyPackMatches` client fn.
- **`src/components/launcher/pack-match-review.tsx`** (new) + **`/apps/icons`
  page** — pack selector, include-hidden/retired toggle, replace-existing toggle,
  per-row apply checkbox + current→proposed previews + confidence badge + reason +
  override `<select>`, Select all/Clear, ConfirmDialog, result summary.
- **Entry points** — "Match icons" button on the Apps header; secondary
  post-import link in the picker's Packs tab (`?pack=<id>`).

### Tests
- `tests/unit/pack-match.test.ts` (20) — match kinds, qualifier stripping both
  ways, no-match, confidence, reasons, selection/overwrite flags, override ref,
  and a `suggestIconKey` regression block.
- `tests/integration/pack-match.test.ts` (11) — service apply behaviour
  (selection, overwrite gate, already-set, unknown icon, app-not-found, variant +
  fallback, mixed counts, unknown pack → null) and route CSRF (403 / same-origin
  200 / 404).
