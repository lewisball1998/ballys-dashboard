# 0003 — SQLite + Drizzle + better-sqlite3

**Status:** Accepted · 2026-06-20

## Context
Need zero-config, file-based persistence suited to a single-container homelab
deploy, with type-safe queries and a committed migration history.

## Decision
SQLite via `better-sqlite3`, with Drizzle ORM. WAL mode + `foreign_keys=ON` +
`busy_timeout`. Migrations generated by `drizzle-kit` into `src/db/migrations`,
committed, and applied at boot (idempotent) by `instrumentation.ts`.

## Consequences
- Data is a single file on a mounted volume (`/app/data`).
- `better-sqlite3` is a native module → marked in `serverExternalPackages` and
  copied into the standalone image; Debian-slim base gives prebuilt binaries for
  amd64 + arm64 (see ADR 0009).
- Only the Architect runs `pnpm db:generate` to keep migration history linear.
- A future move to libSQL/Postgres is possible behind the Drizzle layer if scale
  ever demands it.
